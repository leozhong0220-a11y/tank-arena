# 360坦克对决 联机版 — M4 操作手册

M1:房间与实时移动同步。M2:开炮/血量/重生/击杀榜 + 9 张地图。M3:箱子与道具。
**M4(本版本):音效 + 回合制(先到 10 杀获胜)。**

## M4 新增内容

### 音效(WebAudio 现场合成,无音频文件)
开炮(自己响、别人的炮声小一半)、命中、护盾「叮」、击毁爆炸、捡道具上行三连音、回合胜利号角。
HUD 右上角 🔊 按钮可随时静音。浏览器规定声音必须由用户手势解锁,进房那次点击已经顺便解锁了。

### 回合制
- 右侧名单的 ✕N 就是击杀数,**先到 10 杀获胜**
- 有人达成:全场冻结 6 秒结算(显示获胜者与下一张地图),炮弹和箱子清场
- 自动开新一局:比分清零、所有人满血重生带短暂无敌、**自动换一张新地图**(地图 = 房间码+局数 的哈希,所有人必然一致)

### 同步设计
- **胜负判定走房主权威**:只有房主检查击杀数并广播回合结束,避免各端因偶发丢消息各判各的
- 房主每 5 秒的箱子全量同步现在附带局数,结算期间才进房的人也会在几秒内被校正到正确的地图

## M3 内容回顾

场上会定期刷出「?」木箱(最多 3 个,约 7 秒一个),开过去碾上就捡,种类捡了才知道:

| 道具 | 效果 |
|---|---|
| 加速 | 移速 ×1.6,持续 6 秒(车尾有蓝色速度线) |
| 三连发 | 扇形三发,持续 8 秒(炮口有金色光点) |
| 护盾 | 挡下 1 发炮弹,用掉为止(绿色呼吸光环) |
| 满血修复 | 立刻回满 3 血 |

画布左上角显示自己当前生效的道具和剩余时间;死亡会掉光全部道具。

### 联机同步设计

- **刷箱用房主权威**:加入最早的玩家是房主,负责定时刷箱并广播;房主掉线由下一位自动接任;房主每 5 秒全量同步一次箱子列表,迟到进房的人也能立刻看到场上的箱子
- **捡箱用自己权威**(与命中判定同一哲学):谁碰到谁广播认领。极端情况下两人同帧碰到同一个箱子会都吃到效果——休闲局可接受的取舍
- 道具的视觉状态(盾环/速度线/金点)通过状态快照里的 fx 位标志同步,所有人都能看到你身上挂着什么

## M2.1 地图系统

地图库:经典迷宫 / 十字要塞 / 四象限 / 竞技场 / 蛇形走廊 / 双堡对峙 / 回字城 / 散兵坑 / 风车。

**选图规则:房间码哈希 → 地图编号。**
- 创建房间时房间码是随机的,所以每开一局 = 随机一张图
- 同一个房间码永远对应同一张图,后进房的朋友自动看到同一张地图,不需要任何网络同步
- HUD 会显示当前地图名;想指定某张图,记住对应的房间码重开就行

所有地图都通过了自动验收:出生点不卡墙、全图连通无封死区域。
新地图加在 `config.js` 的 `MAPS` 数组里即可,出生点保险(safePoint)和验收脚本会兜底。

## M2 新增内容

- **开炮**:点击画布 / 按空格 / 手机点「开炮」按钮,炮弹朝炮塔方向飞出,冷却 0.45 秒
- **命中**:每发命中扣 1 血(共 3 血),坦克头顶有血量格,剩 1 血变红
- **击毁与重生**:血量归零爆炸,2.2 秒后在离敌人最远的出生点重生,重生后 1.8 秒无敌(半透明闪烁,开炮立刻解除无敌)
- **击杀榜**:右侧名单每人名字后显示 ✕击杀数

### 联机命中判定的设计(受害者权威)

每台机器只判定"炮弹有没有打中**我自己**",打中了才广播 'ht' 告诉所有人。
好处:永远不会出现"我屏幕上明明躲开了却被判死"——你看到的你就是权威的你。
代价:对方网络很卡时,射手偶尔会看到炮弹"穿过"对方但没掉血,这是公平的取舍。

## 第 1 步:创建 Supabase 项目(约 5 分钟)

1. 打开 https://supabase.com 登录(你做约球 app 的账号就行)
2. New Project → 名字随意(比如 `tank-arena`)→ **Region 选 Oceania (Sydney)** → 数据库密码随便设(用不到)
3. 等 1~2 分钟项目初始化完成
4. 左侧 Settings → API:
   - 复制 **Project URL**(形如 `https://xxxx.supabase.co`)
   - 复制 **anon public** key(很长的一串)
5. 不需要建任何表,不需要改任何设置——我们只用 Realtime 功能

## 第 2 步:填配置

打开 `src/config.js`,把最上面两行换成你刚复制的值:

```js
export const SUPABASE_URL = 'https://xxxx.supabase.co';
export const SUPABASE_ANON_KEY = 'eyJhbGci......';
```

> anon key 写在前端是安全的:它本来就是设计成公开的,而且这个项目没有用到任何数据库权限。

## 第 3 步:本地跑起来

代码用了 ES Module,**不能双击 index.html 直接打开**(浏览器会拦),必须走 http 服务。
在项目文件夹打开 PowerShell,运行:

```
py -m http.server 8000
```

然后浏览器开两个标签页,都访问 http://localhost:8000

- 标签 A:输入昵称 → 创建房间 → 记下 4 位房间码
- 标签 B:输入另一个昵称 → 填房间码 → 加入

## 常见问题

| 现象 | 原因 |
|---|---|
| 提示「连接失败 CHANNEL_ERROR」 | config.js 里 URL 或 key 抄错了,重新核对 |
| 页面全白,F12 控制台报 CORS / module 错误 | 你是双击打开的文件,必须用 `py -m http.server` |
| 对方坦克一卡一卡 | 看名字旁边有没有黄色「…」:有 = 对方网络问题;没有 = 来找我调 INTERP_DELAY |
| 两人互相看不见 | 确认房间码完全一致(4 位字母) |
